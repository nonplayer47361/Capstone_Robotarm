1. 프로젝트 개요

목표

로봇팔이 딸기·블루베리·클레이 구형 모형 등 음식을 인식해 피딩하는 기능을 위한 YOLO 학습용 데이터셋을 팀원이 나눠 효율적으로 수집·라벨링합니다.

2. 도구 구성

📷 collect_webcam.py

웹캠으로 학습용 원본 이미지를 수집합니다. 저장 전 라벨링 단계이므로 raw 폴더에 저장합니다.

  • --prefix: 팀원마다 다른 접두사 → 파일명 충돌 방지
  • --count N: N장 달성 시 자동 종료
  • 한글 경로 완전 지원 (imencode + tofile)
python collect_webcam.py --class-name clay_ball --prefix alice --count 200

🔵 label_clay_ball.py

보라색 구형 클레이 전용 라벨러. HSV 색상 범위로 BBox를 자동 제안합니다.

  • AUTO 탐지: HSV 마스크 → 최대 컨투어 BBox
  • 수동 보완: 드래그로 BBox 직접 그리기
  • n 키: 네거티브 예시 저장 (빈 라벨)
  • --range: 팀원 담당 인덱스 범위 분할
python label_clay_ball.py --images raw/clay_ball --range 0 249

🏷️ annotate_simple.py

여러 클래스를 동시에 라벨링하는 범용 어노테이터. 클래스 패널이 이미지 오른쪽에 표시됩니다.

  • 다중 클래스: 0~9 키로 전환
  • --lock-class: 단일 클래스 고정 (실수 방지)
  • --range: 팀원 담당 범위
python annotate_simple.py --images raw/strawberry --classes strawberry blueberry

⚡ semi_label.py

500장 중 150장만 수동으로 라벨링하고 나머지를 YOLO 모델로 자동 라벨링합니다.

  • Phase 1: 150장 수동 라벨링
  • Phase 2: YOLO 빠른 학습 (~20분)
  • Phase 3: 나머지 자동 라벨 (conf 임계값)
  • Phase 4: 검토 큐 확인/수정
python semi_label.py --images raw/clay_ball --all --manual-count 150

🔀 merge_datasets.py

여러 팀원의 라벨링 결과를 하나의 YOLO 4폴더 데이터셋으로 병합합니다.

  • 3가지 입력 구조 자동 인식 (4폴더/2폴더/평면)
  • 충돌 해결: keep-first / keep-last / most-boxes / interactive
  • --dry-run: 실제 복사 없이 미리보기
  • MD5 기반 결정론적 train/val 분할 유지
python merge_datasets.py --inputs alice/dataset bob/dataset --output merged

🛠️ dataset_utils.py

모든 도구가 공유하는 공통 유틸리티 모듈. 직접 실행 불필요.

  • get_split(): MD5 해시 기반 결정론적 train/val 분할
  • save_labeled(): 4폴더 구조로 이미지+라벨 저장
  • create_yaml(): dataset.yaml 자동 생성
  • print_stats(): 데이터셋 통계 출력

3-3. 비율 유지 (Letterbox) 원리

핵심 원리: 매 프레임 cv2.getWindowImageRect()로 현재 창 크기를 읽어 이미지 영역 크기를 계산합니다. scale = min(ia_w / iw, ia_h / ih)로 비율을 유지하며 축소/확대 후 남는 공간을 어두운 배경으로 채웁니다. 마우스 좌표는 img_x = (canvas_x - off_x) / scale로 이미지 좌표로 변환하므로 창을 어떻게 크기 조절해도 BBox 위치가 정확합니다.

구 방식 (문제)

  • HUD 텍스트가 이미지 위에 직접 그려져 가림
  • 창 크기 조정 시 이미지 늘어남 (비율 파괴)
  • 버튼 클릭 영역이 이미지 좌표계 기준 → 창 크기 달라지면 오프셋 오류

신 방식 (개선)

  • HUD(상단 88px) + 이미지(Letterbox) + 버튼 바(하단 52px) 완전 분리
  • 창 크기 = 캔버스 크기 → 스트레칭 제로
  • 모든 좌표 변환을 _to_img() 한 곳에서 처리

5. 팀 협업 방법

방법 A — --range로 인덱스 분할 (권장)

같은 원본 폴더, 인덱스만 나눔

# 팀원 A: 0~249번 담당
python label_clay_ball.py --images raw/clay_ball --range 0 249 --dataset-dir alice/dataset

# 팀원 B: 250~499번 담당
python label_clay_ball.py --images raw/clay_ball --range 250 499 --dataset-dir bob/dataset

# 병합
python merge_datasets.py --inputs alice/dataset bob/dataset --output merged_dataset

MD5 해시 기반 분할이므로 같은 이미지는 항상 같은 split(train/val)에 배정됩니다.

방법 B — 파일명 접두사 분할 (촬영 단계)

각자 다른 접두사로 촬영 → 병합 시 충돌 없음

# 팀원 A
python collect_webcam.py --class-name clay_ball --prefix alice --count 250

# 팀원 B
python collect_webcam.py --class-name clay_ball --prefix bob --count 250

# 저장 파일명: alice_clay_ball_00000.jpg, bob_clay_ball_00000.jpg (충돌 없음)

주의사항

--val-ratio는 팀원 모두 동일한 값을 사용해야 합니다 (기본 0.2). 서로 다른 val-ratio를 쓰면 같은 이미지가 팀원마다 다른 split에 들어가 병합 후 train/val 오염이 발생합니다.

6. 데이터셋 구조

6-1. YOLO 표준 4폴더 구조

dataset/
├── train/
│ ├── images/   ← 학습용 이미지 (~80%)
│ └── labels/   ← 학습용 라벨 txt (~80%)
├── val/
│ ├── images/   ← 검증용 이미지 (~20%)
│ └── labels/   ← 검증용 라벨 txt (~20%)
└── dataset.yaml   ← 자동 생성

6-2. YOLO 라벨 형식

# 형식: class_id  cx  cy  width  height  (모두 0~1 정규화)
0 0.512000 0.438000 0.124000 0.156000
1 0.720000 0.310000 0.088000 0.095000

6-3. 결정론적 train/val 분할

MD5 해시 기반 분할 원리

def get_split(stem: str, val_ratio: float = 0.2) -> str:
    h = int(hashlib.md5(stem.encode("utf-8")).hexdigest(), 16)
    return "val" if (h % 100) < int(val_ratio * 100) else "train"

파일명이 같으면 누가 언제 라벨링하든 항상 같은 split에 배정됩니다. 팀원 여러 명이 같은 이미지를 라벨링해도 병합 후 오염이 없습니다.
